# Тестовые данные и техники тест-дизайна 

В тестах этого проекта применены базовые техники тестирования данных:

1) **Граничные значения (Boundary Value Analysis)**
- Порт: 1 и 65535 — валидные; 0 и 70000 — невалидные (`validators.test.js`).
- Квота трафика: переполнение и “кап” на максимуме (`usage.test.js`).

2) **Классы эквивалентности (Equivalence Partitioning)**
- Hostname: валидные домены (vpn-1.example.com) vs невалидные (пробелы, дефис в начале) (`validators.test.js`).
- Регион: совпадает (EU/US) vs “не найден” (ASIA) с fallback на лучший по latency (`regions.test.js`).

3) **Негативное тестирование**
- Неверный токен (слишком короткий, запрещенные символы).
- Пустой список серверов в конфиге.
- Неверный host.

4) **Детерминированность / воспроизводимость**
- В функциональном тесте `endToEndEnvConfig.test.js` проверяется предсказуемое поведение при одинаковом env-конфиге:
  выбор сервера по `preferredRegion`, смена `sessionId` на reconnect.

5) **Инварианты системы (для functional tests)**
- После disconnect состояние должно быть `DISCONNECTED`, kill-switch выключен.
- При reconnect должен меняться `sessionId`.
